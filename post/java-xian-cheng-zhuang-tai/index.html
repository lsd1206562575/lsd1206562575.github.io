Java--线程状态

2020-06-24


进入新生状态后，每个线程有自己的工作空间，从主内存中拷贝数据进入自己的工作空间操作。
然后进入就绪状态，只是具有运行的条件，没有分配到cpu，不能马上运行，处于线程的就绪队列中。
系统选择到了就绪的线程后才会进入运行状态。
4种原因进入就绪状态：1.start方法
2.阻塞事件解除，重新进入就绪状态
3.调用yield方法，让出cpu调度
4.jvm将本地线程切换到指定线程
4种原因导致阻塞状态：1.sleep方法 （携带资源）
2.wait方法（不携带资源）
3.join方法， 阻塞指定线程
4.io操作，例如read和write
最后进入死亡状态，原因有两个，分别为线程执行完代码正常结束和线程被强制中止
